# Contributing to Logalytics

Thank you for your interest in contributing to Logalytics! We welcome contributions of all kinds, from bug reports to new features.

## How to Contribute

### Reporting Bugs
If you find a bug, please open an issue on GitHub. Include as much detail as possible, such as:
- A clear, descriptive title
- Steps to reproduce the bug
- Expected behavior vs. actual behavior
- Any relevant logs or screenshots

### Requesting Features
If you have an idea for a new feature, please open an issue to discuss it. We value community feedback and want to make this tool as useful as possible.

### Pull Requests
We welcome pull requests! To submit a contribution:
1. Fork the repository.
2. Create a new branch for your changes.
3. Make your changes and ensure they follow the project's style.
4. Submit a pull request with a clear description of your changes.

## Development Style
- **Shell Scripts**: Use `set -euo pipefail` and include comments for complex logic.
- **JavaScript**: Use modern ES6+ features and keep the code modular.
- **CSS**: Use variables for colors and maintain the dark theme aesthetic.
